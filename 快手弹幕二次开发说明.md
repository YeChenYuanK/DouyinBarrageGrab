# 快手直播弹幕抓取 - 二次开发说明

> 本文档说明如何为 DouyinBarrageGrab 项目添加快手直播弹幕抓取支持，以及它如何与外部 Unity 应用无缝集成。

---

## 1. 架构概述

### 现有架构（抖音）
```

    ↓ (系统代理HTTPS解密)

    ↓ (ProtoBuf解码)

    ↓ (WebSocket广播)

```

```
    ↓
WsBarrageServer
    ↓ (WebSocket广播)
外部应用
```

## 1.5 Unity 兼容模式（无需修改 Unity 项目）

### 背景问题

野套圈等 Unity 项目通过 WebSocket 连接 `ws://127.0.0.1:8888`，期望的消息格式为：

```json
// 评论 (Type=1)
{ "Type": 1, "Data": { "secOpenid":"xxx", "nickName":"xxx", "avatarUrl":"xxx", "content":"xxx" } }
// 礼物 (Type=5)
{ "Type": 5, "Data": { "secOpenid":"xxx", "nickName":"xxx", "avatarUrl":"xxx", "secGiftId":"xxx", "giftNum":1 } }
// 点赞 (Type=2)
{ "Type": 2, "Data": { "secOpenid":"xxx", "nickName":"xxx", "avatarUrl":"xxx", "likeNum":1 } }
```

而 Douyin 内部格式是**嵌套**的：
```json
{ "Content":"xxx", "User":{ "Nickname":"xxx", "HeadImgUrl":"xxx", "SecUid":"xxx" } }
```

### 解决方案

`BarrageMsgPack.ToUnityJson()` 实现了**嵌套数据的自动扁平化**：

- **抖音消息**：将 `{ User: { Nickname, HeadImgUrl, SecUid } }` 展开为 `{ nickName, avatarUrl, secOpenid }`；`GiftId`(long) → `secGiftId`(string)
- **快手消息**：数据直接为扁平格式，无需转换
- `Platform` 字段用于未来扩展（当前 Unity 忽略此字段）

### 关键代码

```csharp
// BarrageMsgPack.cs
public string ToUnityJson()
{
    if (this.Platform == Platform.Kuaishou)
        // 快手：直接返回扁平 Data
        return $"{{\"Type\":{(int)this.Type},\"Data\":{this.Data}}}";

    // 抖音：自动将嵌套 User 展开为顶层字段
    return FlattenDouyinData(this);
}

// KsBarragePusher.cs - 所有消息类型均构造扁平 JSON
var data = new JObject {
    ["secOpenid"] = msg.User?.UserId ?? "",
    ["nickName"]  = msg.User?.Nickname ?? "快手用户",
    ["avatarUrl"] = msg.User?.HeadUrl ?? "",
    ["content"]   = msg.Content ?? ""
};
var pack = BarrageMsgPack.Kuaishou(data.ToString(), PackMsgType.弹幕消息);
```

## 4. 使用方法

### 4.1 通过配置文件启用

**方法一：App.config**
```xml
<add key="kuaishouEnabled" value="true" />
<add key="kuaishouRoomId" value="你的快手号" />
<add key="kuaishouCookie" value="" />
```

**方法二：AppConfig.json**
```json
"kuaishou": {
  "enabled": true,
  "roomId": "你的快手号",
  "cookie": ""
}
```

### 4.2 通过命令行参数启用
```
WssBarrageService.exe --ks-enabled --ks-room-id=你的快手号
```

### 4.3 通过 API 动态连接

```csharp
// 获取 WsBarrageServer 实例后
var roomId = "快手主播的快手号"; // https://live.kuaishou.com/u/xxxxx 中的 xxxxx
await server.KsGrab.ConnectAsync(roomId);
```

---

## 5. 获取快手主播 ID

1. 打开快手直播网页：https://live.kuaishou.com
2. 进入任意主播的直播间
3. 浏览器地址栏 URL：`https://live.kuaishou.com/u/xxxxx`
4. `xxxxx` 部分即为该主播的 `roomId`

---

## 6. 关于 Cookie（可选）

Cookie 不是必须的，但可以提升 API 调用的稳定性和成功率。

**获取方法**：
1. 用 Chrome/Edge 打开 `https://live.kuaishou.com`
2. 登录你的快手账号
3. 按 F12 打开开发者工具 → Network
4. 刷新页面，复制任意请求的 Cookie 头
5. 粘贴到配置文件的 `kuaishouCookie` 字段

## 7. 注意事项

### 7.1 协议稳定性
- 快手可能会不定期更新弹幕协议
- 如果弹幕突然收不到，请检查 real-url 项目是否有最新更新
- https://github.com/wbt5/real-url

### 7.2 错误日志
- 运行日志会输出到程序根目录的 `logs/` 文件夹
- 日志前缀为 `[KS]` 的为快手相关日志

### 7.3 支持的消息类型
| 消息类型 | 说明 |
|---|---|
| `CHAT` | 弹幕消息 |
| `GIFT` | 礼物消息 |
| `LIKE` | 点赞消息 |
| `ENTER` | 进入直播间 |
| `FOLLOW` | 关注消息 |
| `STAT` | 直播间统计（在线人数等） |
| `END` | 直播结束 |

### 7.4 已知限制
- 暂不支持快手私信、粉丝团升级等高级消息
- 快手直播伴侣的代理抓包模式（Hook）暂未实现（需要额外逆向工作）

## 8. 调试技巧

### 启用调试日志
在 `AppConfig.json` 中设置：
```json
"general": {
  "debugMode": true
}
```

### 查看 WebSocket 连接详情
日志中会输出：
- `[KS] 正在连接 WebSocket: wss://...`
- `[KS] WebSocket 已连接 ✓`
- `[KS] 已发送 AUTH 帧`
- `[KS] Binary 消息 CHAT 解析失败`（协议不匹配时）

### 手动测试连接
```csharp
using BarrageGrab.Kuaishou;

var grab = new KsBarrageGrab();
grab.OnChatMessage += (s, e) => {
    Console.WriteLine($"弹幕: {e.Message.Content}");
};
await grab.ConnectAsync("主播快手号");
Console.ReadLine(); // 保持运行
```

## 9. 二次开发建议

### 9.1 协议逆向方向
如果快手协议更新导致无法获取弹幕，建议：
1. 使用 Fiddler/Charles 抓包快手直播页面
2. 搜索 `WebSocket` 连接地址
3. 分析 WS 帧的数据格式
4. 参考 real-url 项目的最新代码

### 9.2 添加新消息类型
在 `ks_message.proto` 和 `KsMessages.cs` 中添加新的消息实体，然后在 `KsBarrageGrab.cs` 的 `DispatchProtoMsg` 方法中添加对应处理分支。

### 9.3 支持快手直播伴侣 Hook 模式
如需支持快手直播伴侣的代理抓包（类似抖音直播伴侣）：
1. 需要分析快手直播伴侣的 HTTPS 弹幕请求
2. 在 `TitaniumProxy.cs` 中添加对应的 Hook 逻辑
3. 参考 `HookBarrage` 方法的实现方式
