## 编译前准备

### 环境要求
- **操作系统**：Windows 7/10/11（.NET Framework 4.6.2 不支持 macOS/Linux）
- **.NET Framework**：4.6.2 或更高
  - 下载地址：https://dotnet.microsoft.com/download/dotnet-framework/net462
- **编译工具**（二选一）：
  1. **Visual Studio 2019/2022**（推荐，含 MSBuild）
     - 下载地址：https://visualstudio.microsoft.com/downloads/
     - 安装时选择「使用 .NET 桌面开发」 workload
  2. **仅安装 MSBuild**：
     - 下载 Visual Studio Build Tools
     - 安装时选择「MSBuild」

### 快速检查

打开 PowerShell 或 CMD，输入以下命令检查环境：

```powershell
# 检查 .NET Framework 版本
reg query "HKLM\SOFTWARE\Microsoft\NET Framework Setup\NDP\v4\Full" /v Release

# 检查 MSBuild
where msbuild
```

## 方法一：一键脚本（推荐）

### 步骤

1. **下载/复制整个项目** 到 Windows 机器
2. **双击运行** `编译为EXE.bat`
3. **按提示选择** 编译模式（默认 Release）
4. 等待编译完成，EXE 文件在 `Output/WssBarrageServer.exe`

### 脚本功能
- ✅ 自动检测 .NET Framework
- ✅ 自动恢复 NuGet 包
- ✅ 自动编译 Release/Debug
- ✅ 自动清理调试文件（Release 模式）
- ✅ 自动复制配置文件

## 方法二：Visual Studio 图形界面

### 步骤

## 方法三：命令行 MSBuild

```powershell

nuget restore BarrageService.sln

msbuild BarrageService.sln /p:Configuration=Release /p:Platform="AnyCPU"
```

## 常见问题

### Q: 编译报错「找不到 NuGet 包」
**A**: 确保网络连接正常，运行 `nuget restore BarrageService.sln` 手动恢复包

### Q: 报错「缺少 Costura.Fody」
**A**: 运行 `nuget install Costura.Fody -Version 5.7.0` 或在 Visual Studio 中右键「管理 NuGet 包」→ 还原

### Q: 找不到 MSBuild
**A**: 安装 Visual Studio Build Tools，选择「MSBuild」组件

### Q: 想同时修改源码
**A**: 克隆仓库到 Windows，用 Visual Studio 打开 `BarrageService.sln` 即可调试
